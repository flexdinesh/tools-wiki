# Plan: Integrate Fira Code Font via Fontsource

**Date**: 2026-05-04

## Summary

Replace default Starlight fonts with Fira Code across the entire site (body text + code) using Fontsource. Import Regular (400) and Semibold (600) weights. Disable programming ligatures.

## Files Changed

| # | File | Action | Details |
|---|------|--------|---------|
| 1 | `package.json` | Edit | Add `@fontsource/fira-code` to `dependencies` |
| 2 | `src/styles/custom.css` | Edit | Add `@import` for weights 400 and 600; set `--sl-font` and `--sl-font-mono`; disable ligatures |
| 3 | `pnpm-lock.yaml` | Auto-update | Regenerated by `pnpm install` |

## Decisions

1. **Scope: Body + code** — Both `--sl-font` and `--sl-font-mono` set to Fira Code. Entire site renders in monospace.
2. **Weights: 400 + 600** — Regular + Semibold. Minimum viable set for readable body text and functional bold highlighting in code blocks.
3. **Ligatures: Disabled** — `font-feature-settings: "liga" off, "calt" off;`. Explicit override to prevent browser defaults.

## Tradeoffs

- **Readability**: Fira Code is optimized for programming, not extended reading. Body text in monospace may feel dense to non-developer visitors.
- **Bundle size**: Two weights ≈ 160KB of font files. Acceptable for the feature.
- **Future migration**: If body text readability becomes an issue, splitting to a proportional body font later requires only changing `--sl-font` (not `--sl-font-mono`).

## Execution Steps

1. Run `pnpm add @fontsource/fira-code`
2. Update `src/styles/custom.css` with font imports and CSS variable overrides
3. Run `pnpm dev` to verify fonts load in both light and dark modes
